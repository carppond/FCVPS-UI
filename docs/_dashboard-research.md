# Dashboard 设计调研

> 调研目的：为"拾光VPS"找到真正好看的 Dashboard 参考，避免重蹈传统两栏布局 / Bento Grid 的覆辙。
> 调研方式：WebSearch + WebFetch 实际抓取各产品 GitHub README、官网、博客文章。

---

## 1. Nezha v2（哪吒面板 v1/v2）

- **布局**：前端后端分离架构。官方默认前端为社区项目 nezha-dash（hamster1963 维护）。主页为服务器列表，每台服务器以卡片或行展示，显示 CPU%、内存%、网络上下行、在线/离线状态。支持深色/浅色主题切换。管理后台 `/dashboard` 路径单独存在，用于配置告警、服务检测、任务等，风格偏传统后台表单。
- **视觉风格**：nezha-dash 社区版（Next.js 实现）走现代简洁路线，卡片干净，支持 dark 主题。信息密度中等，移动端体验改善明显（V1 改进点之一）。配色以中性灰/白为主，accent 颜色可通过主题覆盖。
- **亮点**：社区主题生态活跃（有 MDUI 主题、那爪 Nazhua 主题等），前台用户界面与管理后台分离使定制门槛降低；新版放弃 OAuth 改用本地账号，部署更简单。
- **不足**：默认主题视觉辨识度一般，缺乏品牌感；管理后台（Go 原生模板）与前台（Next.js）割裂感强；V1 主版本信息仍以表格/行式列表为主，没有真正意义上的"总览仪表盘"——只有服务器清单，没有聚合指标卡。

## 2. 妙妙屋（miaomiaowu）

- **布局**：Clash 订阅管理工具，不是服务器监控面板。主要功能模块：流量监控面板（实时统计图表）、订阅管理列表（节点 + 外部订阅）、代理分组配置（可拖拽排序）、用户管理。页面采用功能模块横向 Tab 或侧边导航切换。
- **视觉风格**：支持亮色/暗色模式，中等信息密度，有可折叠/可展开的节点列表，配置界面为表单+可视化编辑器混合风格。整体偏工具性，不追求视觉冲击。
- **亮点**：可拖拽的代理分组编辑是核心交互亮点；多订阅聚合 + 流量统计呈现在同一界面，省去切换工具的麻烦。
- **不足**：专注订阅管理，无服务器探针指标；UI 偏功能导向，视觉吸引力有限；对非技术用户上手门槛不低（配置项多）。

## 3. Beszel

- **布局**：两层结构。**Hub 主页**（多服务器总览）：以列表或行式布局展示所有被监控系统，每行显示服务器名称 + 实时状态（绿/黄/红指示灯）+ CPU%/内存%/网络带宽等数值；点击进入**System 详情页**：按指标分组展示多条折线图（CPU、内存、磁盘 I/O、网络），Docker 容器各分配颜色在同一图表中叠加显示，可按容器过滤。无传统左侧导航栏——顶部或极简侧边操控即可完成全部操作。
- **视觉风格**：深色主题为主（light 模式亦支持），现代简洁，低信息噪声。折线图采用多色彩编码区分 Docker 容器，视觉上清晰。字体干净，卡片间距舒适，不拥挤。整体气质接近 Vercel/Linear 的 SaaS 监控工具。
- **亮点**：信息密度的"甜点"——不像 Netdata 那样海量指标轰炸，也不像简陋脚本那样只有数字；颜色编码 Docker 服务折线图是真正实用的设计决策；极轻量（Go + PocketBase），UI 却不像轻量工具。
- **不足**：总览页信息聚合能力有限，只有行式列表，没有跨服务器的汇总指标卡（如"所有服务器平均 CPU"）；缺少订阅/代理管理相关模块，与 FCVPS-UI 的业务边界差异较大。

## 4. Uptime Kuma

- **布局**：经典双栏。左侧固定侧边栏：监控列表（按分组 Group 折叠展开），每行显示服务名 + 状态圆点（绿/橙/红）+ 近 90 天 ping 条形图（竖线阵列，绿=正常/红=故障）+ 响应时间数值。右侧主区域：选中某个监控后显示详情——历史响应时间折线图、事件日志、SSL 证书信息等。顶部全局状态栏。
- **视觉风格**：Bootstrap 5 + Vue 3，两套主题（深/浅）。深色模式下背景 #1b1b1b 系深灰，卡片略浅（#242424），accent 默认绿色。信息密度中等偏高（左侧列表可能很长）。ping 条形图是视觉标志性元素——90 格竖线一眼看出历史健康度。
- **亮点**：ping 历史条形图（90 天 uptime bar）是设计精髓，用极小空间传达丰富历史信息；状态颜色系统简洁直觉（绿/黄/红）；社区有大量 CSS 主题扩展，可改造成毛玻璃风格。
- **不足**：核心是 Uptime 监控，不涵盖 CPU/内存/流量等系统指标；纯左侧列表布局在服务器数量多时容易变成"一列滚动文字墙"；公开状态页与私有管理界面风格不统一。

## 5. Clash Verge Rev

- **布局**：桌面客户端（Tauri 2 + TypeScript），非 Web 面板。左侧窄导航栏（图标+文字，约 6-8 个入口：主页/代理/订阅/规则/连接/日志/设置）；右侧主区域随导航切换。**主页**：可自定义的状态卡片区（连接数、上下行流量、延迟）。**代理页**：代理分组列表 + 节点延迟可视化（格子或列表），支持搜索过滤。
- **视觉风格**：现代 UI，深色和浅色双模式，支持自定义主题色（accent color picker）和自定义 CSS 注入。整体干净，间距舒适，接近 macOS 原生应用审美。代理组的延迟数字配合颜色（绿<100ms/黄100-200ms/红>200ms）形成快速视觉扫描体验。
- **亮点**：代理节点延迟的颜色编码+数字显示非常实用；自定义 CSS 支持使社区能做出接近 Raycast 风格的皮肤；Tauri 轻量运行，启动快；支持代理组图标自定义，增加视觉辨识度。
- **不足**：是桌面客户端非 Web 面板，架构参考价值有限；主页 Dashboard 内容相对简单，核心交互在代理页而非首页；缺少服务器探针监控能力。

## 6. 1Panel

- **布局**：典型 SaaS 管理后台风格。顶部横向 Tab 导航（应用商店/网站/数据库/容器/主机/设置）或左侧侧边栏（版本依赖）；主内容区为对应功能模块。**主机监控**部分：CPU/内存/磁盘/网络折线图，以 Grid 排列在同一页面，实时刷新。容器管理：表格式列表（Name/Status/Image/Port）+ 操作按钮。最近版本引入 AI/LLM 管理模块（Ollama 等）。
- **视觉风格**：现代 SaaS 风格，以浅色为主（深色模式在后续版本支持）。配色专业中性，卡片圆角，表格斑马纹。信息密度中等，不拥挤但也不空洞。技术人群友好，操作逻辑清晰。
- **亮点**：功能覆盖范围最广的开源面板之一（网站+数据库+容器+主机一体）；主机监控的多图表 Grid 布局信息完整；应用商店概念降低部署门槛；界面干净无广告（对比宝塔）。
- **不足**：视觉风格偏"企业内部系统"，缺乏个人项目的温度；总览主页不突出，信息聚合度弱（分散在各 Tab 下）；配色较平淡，accent 不鲜明；深色模式支持相对滞后。

## 7. 宝塔面板（aaPanel / BT Panel）

- **布局**：传统双栏：左侧固定导航栏（网站/FTP/数据库/软件商店/文件/监控/防火墙/面板设置等约 15 个入口，带图标）；右侧主区域。首页总览：CPU/内存/磁盘/网络的仪表盘式小图（半圆 gauge）+ 近期统计数字 + 告警提示 + 近期操作日志。
- **视觉风格**：老式 Bootstrap 风格（宝塔国内版广告多，aaPanel 国际版稍简洁），默认浅色，有可选深色皮肤。导航图标配色偏卡通，整体视觉杂乱。首页 gauge 仪表盘偏 2015 年 UI 审美。
- **亮点**：信息量大，首页一屏看全主机核心指标；软件商店和一键部署是真实用户需求；gauge 仪表盘对非技术用户直觉友好。
- **不足**：UI 老旧，设计语言停留在 2015-2018 年代；国内版广告轰炸严重；左侧导航入口过多导致认知负担重；半圆 gauge 比折线图信息量低（只有当前值，无历史趋势）；整体视觉凌乱，与现代深色简洁风格相去甚远。

## 8. Portainer

- **布局**：左侧固定导航栏（Home/App Templates/Stacks/Containers/Images/Networks/Volumes/Settings），导航采用深色背景（#232D35 深蓝灰）+ 高亮 accent（蓝色）。**Home 页**：所有 Docker 环境的健康总览，每个环境一块卡片（Endpoint 名 + 状态 + 容器数/镜像数/Volume 数）。**Dashboard 页**（进入某环境后）：多个 Summary Tile 展示 Stacks 数、Services 数、Containers 数（运行/停止/不健康分色显示）、Images 磁盘占用、Volume 数、Network 数。
- **视觉风格**：专业深色导航栏 + 浅色主区域，或全深色模式（2.9.0 后支持）。Summary Tile 为方形数字卡，简洁有力。信息密度适中，容器状态分色（绿=运行/灰=停止/红=不健康）直观。整体类似企业 DevOps 工具审美。
- **亮点**：Summary Tile 多指标数字卡设计简洁有力，一屏看全环境概况；状态色彩系统统一；环境切换（多 Docker host）设计清晰；深色导航 + 浅色内容的分区对比清晰。
- **不足**：主要针对容器管理，无法直接用于代理/订阅管理；浅色主区域与现代深色全局风格有割裂感；Summary Tile 仅数字，缺少趋势/历史信息；导航条目细碎，初次使用找不到入口。

---

## 9. Grafana

- **布局**：左侧折叠导航栏（Home/Dashboards/Explore/Alerting/Connections/Admin，图标式，可 dock/undock）；主区域为自由 Grid 画布，由 Panel 构成（Panel = 数据查询 + 可视化组合）。Panel 可自由拖拽、调整大小，支持"行"（Row）折叠分组。典型监控仪表盘：顶部一行 Stat Panel（单数值大字+趋势箭头）→ 第二行时序折线图 → 第三行 Gauge/Bar Gauge → 底部 Table/Log Panel。时间范围选择器在右上角（Last 6h / Last 24h 等）。
- **视觉风格**：深色为主（#111217 背景，Panel 背景 #181B1F，accent 橙色 #FF7800 或可自定义）。设计语言偏数据可视化工具而非管理后台：强调图表、颜色映射、数据密度。支持多种颜色主题，深色模式视觉权重感强。信息密度高但通过 Panel 边界保持秩序感。
- **亮点**：自由 Grid 布局最灵活，任意信息密度均可实现；Stat Panel（大数字+单位+趋势箭头）是监控场景最高效的信息载体；颜色 threshold 系统（Stat Panel 背景随数值变色）极为直观；折线图的时间范围滑动控制是标杆交互。
- **不足**：灵活度即复杂度，搭出好看仪表盘需要较高学习成本；无订阅/代理管理能力，纯监控工具；开箱即用的 Home Dashboard 并不好看，依赖用户配置；对普通用户太"工程师"化，上手门槛高。

---

## 10. Shadcn Dashboard 模板

- **布局**：官方 shadcn/ui Dashboard 示例：左侧侧边栏（品牌名 + 分组导航：Home/Documents/Settings）+ 主内容区。主区上部：4 个横排指标卡（Total Revenue / New Customers / Active Accounts / Growth Rate），每卡含大数字 + 增减百分比趋势标注。中部：一个大折线图（6 个月访客趋势，含时间范围 Tab 筛选）。下部：数据表格（带分页、拖拽排序、列状态过滤）。社区扩展（如 shadcn-admin）额外提供 10+ 页面：认证、用户管理、权限、设置等。
- **视觉风格**：Next.js / Vite + Tailwind CSS 4 + Radix UI，中性色调为主，深浅两模式均支持，设计极度克制（无多余装饰），依赖排版和间距建立层次。指标卡圆角大、padding 充足、数字字号突出、趋势百分比配颜色（绿色上升/红色下降）。整体气质接近 Vercel Dashboard 或 Linear。
- **亮点**：4 指标卡 + 1 大图 + 1 表格的三段式结构已成现代管理后台的"黄金模板"，信息层次最清晰；Shadcn 组件天然支持暗色，可无缝叠加 Raycast 色板；社区模板质量高（12 套专业主题），拿来即改。
- **不足**：默认模板偏通用商业后台风格，与服务器监控/VPS 场景有距离需要定制；图表库需另接（Recharts/Tremor），非开箱即用的实时监控图表；对于技术指标展示（CPU 曲线、流量波形），需要较多定制工作。

## 结论：推荐方向

综合调研，结合"拾光VPS"的定位（Raycast 深色风 + 红橙 accent + 技术用户 + 核心信息：探针指标/订阅状态/节点健康/流量/最近事件），推荐以下 3 个方向，按推荐度排序：

---

### 方案一（最推荐）：Beszel × Shadcn 混合——"指挥中心"横幅式布局

**参考来源**：Beszel 的双层层级（总览→详情）+ Shadcn 的指标卡三段式 + Grafana 的 Stat Panel 大数字

**核心思路**：顶部 4 个大数字 Stat 卡（全局汇总）→ 中部三列探针卡（每台服务器一格）→ 底部两栏（订阅状态 + 最近事件流）

```
┌─────────────────────────────────────────────────────┐
│  ← 顶部 Banner：4 个 Stat 卡（等宽，红橙 accent）   │
│  [ 在线服务器 12/14 ]  [ 平均CPU 23% ]  [ 今日流量 ] [ 告警 2 ] │
├─────────┬───────────────────────────────────────────┤
│         │  探针列表（每台服务器一张卡，3列网格）        │
│  侧边栏  │  ┌──────┐ ┌──────┐ ┌──────┐              │
│  (图标)  │  │Server│ │Server│ │Server│              │
│         │  │CPU ██│ │CPU ██│ │CPU ██│              │
│  导航    │  │MEM ██│ │MEM ██│ │MEM ██│              │
│  探针    │  │NET ↑↓│ │NET ↑↓│ │NET ↑↓│              │
│  订阅    │  └──────┘ └──────┘ └──────┘              │
│  节点    │  ...（可滚动）                             │
│  流量    ├─────────────────┬─────────────────────────┤
│  事件    │  订阅状态（列表）  │  最近事件 / 告警流        │
│  设置    │  ● 套餐A 到期6天  │  ⚠ server3 CPU>90%     │
│         │  ● 节点组 B 12节点│  ✓ server2 恢复在线      │
└─────────┴─────────────────┴─────────────────────────┘
```

**视觉执行**：背景 #0D0D0F（接近 Raycast），侧边栏毛玻璃 backdrop-blur，探针卡 border-[#2A2A2E] + hover 时红橙描边，Stat 卡 accent 文字用 #FF5F57（Raycast 红）/ #FF9A3C（橙）。

---

### 方案二（次推荐）：Uptime Kuma 左列 × Grafana Stat Panel——"双区域时序"布局

**参考来源**：Uptime Kuma 的左侧服务器列表 + Grafana 的 Stat Panel + 折线图区域

**核心思路**：左侧固定列表（服务器/订阅/节点快速扫描）+ 右侧主区域随选中项展示详细时序图

```
┌─────────┬───────────────────────────────────────────┐
│ 🔴 srv1 │  服务器详情：server1                        │
│ 🟢 srv2 │  ┌─────┐ ┌─────┐ ┌─────┐ ┌─────┐         │
│ 🟢 srv3 │  │ CPU │ │ MEM │ │ NET │ │DISK │  Stat    │
│ ─────── │  │ 23% │ │ 67% │ │12MB│ │ 45% │  Cards   │
│ 订阅A   │  └─────┘ └─────┘ └─────┘ └─────┘         │
│ 订阅B   │                                            │
│ ─────── │  CPU 时序折线图（24h）────────────────────  │
│ 节点组1  │  ████████████████████████████████        │
│ 节点组2  │                                            │
│         │  网络 IN/OUT 面积图（24h）─────────────────  │
│         │  ████████████████████████████████        │
│         │                                            │
│  [告警]  │  最近事件 ─────────────────────────────── │
└─────────┴───────────────────────────────────────────┘
```

**视觉执行**：左列宽 240px，深色 + ping 竖线阵列（仿 Uptime Kuma 90天条形图）；右侧 Stat Card 背景随值变色（CPU>80% 卡片变橙/红）。

---

### 方案三（备选）：Portainer Summary Tile 变体——"磁贴总览"全局视角

**参考来源**：Portainer Summary Tile + 1Panel 主机图表 Grid

**核心思路**：没有左侧列表，全页面用不等大磁贴组合，大磁贴=最重要信息，小磁贴=次要数字，点击磁贴下钻详情。

```
┌──────────────────────────────────────────────────────┐
│ [ 拾光VPS ]                              [ 设置 / 用户 ] │
├────────────┬────────┬────────┬────────────────────────┤
│            │ 在线   │ 告警   │                         │
│  探针总览   │ 12/14  │  2条   │   24h 流量面积图         │
│  （折线图） │        │        │   ██████████████████  │
│            │ CPU均  │ 流量均  │                         │
│            │  23%   │ 45MB/s │                         │
├────────────┴────────┴────────┼────────────────────────┤
│  订阅健康（3张小卡横排）        │  最近告警/事件（列表）   │
│  套餐A ✓ | 套餐B ⚠ | 套餐C ✓  │  - srv3 CPU spike      │
├─────────────────────────────┼────────────────────────┤
│  节点延迟热力格（代理节点组）   │  到期提醒 / 操作快捷入口  │
│  US-LA ●12ms HK ●8ms ...   │  [ 续费 ] [ 测速 ]       │
└─────────────────────────────┴────────────────────────┘
```

**视觉执行**：不等大磁贴间 gap 16px，圆角 12px，背景毛玻璃；热力格用颜色深浅表示延迟（类 GitHub contribution 图）；整体无硬分栏，信息自然流动。此方案视觉最独特，但开发复杂度最高。
