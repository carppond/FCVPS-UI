# 探针告警规则 (Agent Alerting)

为探针设置阈值,当 **CPU / 内存 / 磁盘使用率**超过阈值,或探针**离线**时,经你已配置的[通知渠道](./notification-setup.md)(Telegram / 邮件 / Webhook 等)自动告警。

后台引擎每 **60 秒**评估一次所有启用的规则;命中后发送一次告警,并在「冷却时间」内不再重复发送(防刷屏)。

**进入**:
- Web:左侧栏 **告警规则**
- 移动端:**更多 → 告警规则**

---

## 新建一条规则

| 字段 | 说明 |
|------|------|
| **规则名称** | 便于识别,如「Web 服务器 CPU 过高」 |
| **目标探针** | 选「全部探针」对你名下所有探针生效;或指定单个探针 |
| **指标** | `CPU 使用率` / `内存使用率` / `磁盘使用率`(百分比),或 `探针离线` |
| **阈值 (%)** | 指标 ≥ 此值即触发(`离线` 类型忽略此项) |
| **持续时间(秒)** | `0` = 立即触发;`>0` = 需在该时间窗口内**持续**超阈值才触发(离线类型表示「离线超过这么久」才告警) |
| **冷却(秒)** | 一次告警后,至少间隔这么久才会对同一(规则,探针)再次告警。默认 `3600`(1 小时) |
| **启用** | 关闭后规则保留但不评估 |

---

## 工作原理

- **即时规则**(持续=0):取该探针最近一次上报的指标值,超阈值即告警。
- **持续规则**(持续>0):取窗口内全部样本,**每一个**都超阈值才告警——避免瞬时尖峰误报。
- **离线规则**:探针状态为离线,且离线时长 ≥ 持续时间时告警(持续=0 则一离线就告警)。
- **冷却**:按 (规则 ID + 探针 ID) 独立计时,所以多探针规则对每台机器各自冷却。

> 告警走的是统一通知系统:确保你在[通知](./notification-setup.md)里**已配置渠道并勾选接收**——告警事件类型为 `probe_alert`。没有可用渠道时,规则仍会评估,但你收不到推送。

---

## 示例

- **CPU 持续 5 分钟超 90% 才告警,每小时最多一次**:指标 `CPU 使用率`、阈值 `90`、持续 `300`、冷却 `3600`。
- **任一探针离线 3 分钟即告警**:目标 `全部探针`、指标 `探针离线`、持续 `180`。
- **磁盘一旦超 95% 立即告警**:指标 `磁盘使用率`、阈值 `95`、持续 `0`。

---

## 常见问题

**没收到告警?**
1. 通知渠道是否已配置且勾选了接收?(告警类型 `probe_alert`,可在通知页确认渠道已启用)
2. 规则是否「启用」?目标探针是否在上报数据?(无数据的指标不会触发)
3. 是否还在冷却期内?同一条件在冷却时间内只发一次。

**告警太频繁?** 调大「冷却」,或给指标规则设「持续时间」过滤瞬时尖峰。
