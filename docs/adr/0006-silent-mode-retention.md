# ADR 0006: 保留妙妙屋"静默模式"独家工程亮点

## Context

- 调研报告（docs/_research-competitors.md 第二节横向对比表）显示，"静默模式（未授权返 404）"在 10 个候选项目中**仅妙妙屋有**，是业界稀缺特性。
- 自托管 Clash 面板 / 探针类服务暴露公网必然会遭遇扫描器（如 Censys、ZoomEye 关键词扫，以及黑产针对 v2board / Xboard 等已知后台的爆破）。
- 普通防御（限速 / 验证码）暴露"此处是某 X 后台"，反而成为攻击者目标。静默模式直接返 404，扫描器记录的会是"该 IP 无服务"，从被动防御升级为"主动隐身"。
- 该特性已在妙妙屋实际部署中被验证有效，是其差异化记忆点。

## Decision

**保留并强化静默模式**为拾光VPS v1 必备特性：

1. **默认开启**：首次部署即处于静默模式；可由 admin 在设置面板关闭（不推荐，但保留逃生舱）。
2. **生效范围**：所有未带合法授权（Cookie / API token / agent token）的 HTTP 请求一律返回 `404 Not Found`，包括：
   - 错误密码登录（避免账号枚举）
   - 已知路径但未授权（如 `/api/...` 不带 token）
   - 未知路径（如 `/admin`、`/login` 等常见后台路径）
3. **唯一例外**：
   - `/healthz`（K8s / 监控用，可由 admin 关闭）
   - 短链系统的 `fileCode+userCode` 复合路径（必须可被客户端访问）
4. **登录页 / 静态资源**：通过特殊 path 前缀（默认 `/_app/<random-32hex>/`，部署时一次性生成）暴露；非该前缀全 404。该前缀的值在 `.env` 或首次启动日志中提示给 admin。
5. **响应特征清理**：自定义 `Server` header、移除"X-Powered-By"、404 body 与 nginx default page 完全一致。

## Consequences

**正面**：
- 扫描器无法发现服务存在，业界 OSINT 工具无法识别为"拾光VPS"，防御等级显著拉升。
- 与 ADR 0002 多用户体系协同：未登录的"另一个用户"也走 404，避免跨用户嗅探。
- 是面对小机场主用户群的差异化卖点（同类项目均无）。

**负面 / 待办**：
- 用户首次部署看不到"漂亮的登录页"，需要先记下 `_app/<random>/` 前缀；这是 by design 的代价。
- 监控告警基于 HTTP 状态码时易误报（看到大量 404）；文档需明示。
- Telegram Bot webhook 路径默认也走静默模式；需要单独 token 校验白名单。

**替代方案为何放弃**：
- **关闭静默模式（妙妙屋之外所有项目的现状）**：丢失差异化，遭扫描器骚扰。
- **WAF / Cloudflare 前置防护**：依赖第三方，且 Cloudflare 不阻止"针对你 IP 的扫描"；治标不治本。
- **HTTP basic auth 前置**：暴露"此处有保护"，反而吸引爆破。
