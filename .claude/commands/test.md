## 执行步骤

### 1. 安装测试依赖
按技术栈选主流框架（如未安装）：
- Node：vitest / jest + supertest
- Python：pytest + httpx
- Go：testing + testify
- Rust：内置 + tokio-test

### 2. 单元测试

为每个模块的核心函数/方法写测试，覆盖：正常路径 + 边界 + 异常。
测试文件按项目惯例放置（`__tests__/`、`tests/`、`*_test.go` 等）。

运行后写入 `docs/07-unit-test-report.md`：

- 总用例数：xx
- 通过：xx | 失败：xx | 跳过：xx
- 覆盖率：xx%（如可用）

## 测试用例清单

| 用例 | 状态 | 说明 |
|------|------|------|

## 失败用例详情
（如有失败，记录错误与原因分析）

还需要补充的部分。

### 3. 集成测试

验证模块/各端协作。重点：
- API 接口的请求/响应格式与 `docs/04-api-contract.md` 一致
- 状态码 / 错误码与契约一致
- 核心业务流程端到端可走通（创建 → 查询 → 更新 → 删除）
- 模块间数据传递正确

按平台启动服务（`npm start` / `go run` / `python -m ...`），运行集成测试。如果服务起不来，记录错误信息，跳过运行阶段，在报告里标注。

写入 `docs/08-integration-test-report.md`：

## 集成测试概览
