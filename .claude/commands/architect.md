## 输出要求

请输出架构设计文档，保存到 `docs/03-architecture.md`，包含：

### 1. 技术选型
- 语言/框架/运行时的选择及理由
- 关键依赖库及版本
- 构建和包管理工具

### 2. 项目结构
用目录树展示完整的项目结构，标注每个目录/文件的用途。

### 3. 模块划分
- 核心模块列表及职责
- 模块之间的依赖关系（用文字或 ASCII 图）
- 每个模块的公共接口

### 4. 数据模型
- 核心数据结构/类型定义
- 数据库 Schema（如需要）
- 数据流向

### 5. 接口设计
- API 接口列表（如果有）
- 模块间的内部接口
- 输入输出格式

### 6. 关键设计决策
- 做了哪些架构取舍
- 为什么这样选择（trade-off 分析）

## 规则
- 技术选型要考虑团队熟悉度和生态成熟度
- 优先选择简单直接的方案，避免过度设计
- 目录结构要具体到文件级别，开发可直接照搬
- 模块边界要清晰，确保可以独立开发
- 必须明确标注哪些是"共享文件"（入口文件、配置文件、类型定义），哪些是"模块文件"，方便后续 Tech Lead 拆分任务
- 如果存在 `docs/02-ui-design.md`，且它指定了视觉风格 + Design Tokens，则前端 UI 库选型须能承载该风格（如 Glassmorphism / Brutalism 优先 Tailwind + headless UI；Material 3 用 MUI；iOS HIG 用 SwiftUI；标准 SaaS 用 shadcn/ui）
